# 实施计划

- [ ] 1. 重构文章发布目标模型，解除发布流程对单一 `agent.account_id` 的硬绑定
  - 梳理文章发布、草稿同步、历史查询中对 `agent.account_id` 的直接依赖
  - 为发布请求补充显式目标公众号参数的入口，并保留默认值兼容逻辑
  - 调整服务层接口签名，使发布目标由调用参数决定而不是由 Agent 唯一决定
  - _需求：4.1、4.2、4.5、6.5_

- [ ] 2. 建立文章-公众号独立发布关系的数据模型与持久化逻辑
  - 新增文章与公众号之间的独立发布关系记录，用于保存草稿 ID、发送状态、错误信息和更新时间
  - 实现首次同步创建关系、再次同步更新关系的持久化流程
  - 为历史查询与结果展示提供按文章和公众号维度读取关系记录的能力
  - _需求：3.4、3.5、3.6、4.3、4.4_

- [ ] 3. 改造发布与同步 API，支持单篇文章指定一个或多个目标公众号
  - 修改文章发布/同步接口的请求参数校验，支持接收公众号 ID 列表
  - 增加“未选择公众号”“公众号无权限/不可用”等边界校验
  - 返回结构按公众号维度输出独立结果，避免单一全局状态覆盖
  - _需求：3.1、3.2、6.1、6.2、6.3_

- [ ] 4. 实现多公众号独立执行编排，确保单个公众号失败不影响其他公众号结果
  - 将单次发布拆分为按公众号粒度执行的独立处理单元
  - 为每个公众号分别执行封面处理、正文图片处理、草稿创建/更新和结果回写
  - 统一汇总每个公众号的成功、失败、草稿 ID 与失败阶段信息
  - _需求：3.1、3.3、5.1、5.4、6.1_

- [ ] 5. 完善正文图片素材落库逻辑，使正文相关媒体在素材库中可见
  - 在正文图片上传/重写流程中为每张图片生成独立素材记录
  - 保存原始来源地址、微信托管地址、关联文章、关联公众号、上传状态与失败原因
  - 确保同一篇文章多张正文图片分别记录，而不是仅保留单一素材结果
  - _需求：1.1、1.3、1.4、1.5、1.6_

- [ ] 6. 改造素材库查询与展示接口，纳入正文图片素材视图
  - 扩展素材库数据源，使正文图片素材与封面素材可统一检索
  - 支持按文章、公众号、素材状态等维度过滤正文素材记录
  - 在返回结果中体现素材来源、上传时间、当前状态和错误信息
  - _需求：1.2、1.3、1.4、6.4_

- [ ] 7. 按公众号隔离微信媒体处理，避免复用错误的媒体标识
  - 检查并修正封面与正文图片上传流程，确保使用当前目标公众号的访问令牌
  - 禁止跨公众号复用仅对单一公众号有效的 `media_id`、`thumb_media_id` 或图片 URL 处理结果
  - 在正文图片失败时阻断该公众号草稿提交，并保留独立错误详情
  - _需求：5.1、5.2、5.3、5.4、5.5_

- [ ] 8. 为本地 skill 增加面向远端环境的受控测试参数
  - 更新本地 skill 的输入与调用说明，支持显式传入远端服务地址、目标公众号和测试模式
  - 将 skill 调用改为走正式接口参数，而不是通过修改数据库绑定关系进行测试
  - 输出测试结果摘要，包含目标公众号、失败阶段、错误信息和关键返回数据
  - _需求：2.1、2.2、2.3、2.4、2.5_

- [ ] 9. 为多公众号发布与正文素材链路补充自动化测试
  - 增加服务层测试，覆盖首次同步、重复同步、多公众号部分失败、未选择公众号和无权限公众号等场景
  - 增加正文图片素材记录测试，覆盖本地媒体、远程图片、base64 图片及失败回写
  - 增加接口测试，验证按公众号维度返回独立结果和素材库过滤能力
  - _需求：1.1、1.4、2.3、3.3、5.4、6.1、6.2、6.3_

- [ ] 10. 补充兼容迁移与回归验证代码
  - 为历史已存在的单公众号草稿记录提供兼容读取或迁移脚本
  - 确认旧接口在未传目标公众号参数时仍能按默认逻辑运行
  - 为多公众号新流程与旧单公众号流程分别编写回归验证用例，降低上线风险
  - _需求：3.5、4.5、6.5、6.6_
