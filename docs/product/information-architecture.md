# Agent Publisher 信息架构设计

> 最后更新：2026-04-04

## 设计原则

1. **路径最短**：从看到热点到开始创作，步骤最少
2. **功能收敛**：22 个页面 → 4 个核心入口 + 设置区
3. **渐进披露**：核心路径极简，高级功能按需展开
4. **配置与生产分离**：设置类页面不干扰创作流程
5. **零配置可用**：内置默认 Agent，新用户无需任何配置即可创作
6. **创作与编辑统一**：不再区分"AI 生成"和"文章编辑"，统一在创作控制台完成

## 导航结构

```
┌────────────────────────────────────────────────────────────┐
│  🔥 Agent Publisher          [Credits: 450] [头像▼]        │
├────────────┬───────────┬───────────┬───────────┬──────────┤
│  🔥 发现   │  ✍️ 创作   │  📦 内容   │  📊 数据   │  ⚙️     │
│  /discover │  /create  │  /content │  /stats   │ /settings│
└────────────┴───────────┴───────────┴───────────┴──────────┘
```

### 4 个核心页面

| 页面 | 路由 | 定位 | 核心动作 |
|------|------|------|---------|
| **发现** | `/discover` | 生产入口 | 浏览热点、筛选、一键创作、收藏素材 |
| **创作** | `/create` | 生产核心 | 选 Agent + 模板、AI 生成、编辑、预览 |
| **内容** | `/content` | 资产管理 | 草稿箱、已发布、发布操作 |
| **数据** | `/stats` | 效果追踪 | Credits 用量、生成统计（v2.0） |

### 设置区（⚙️ 齿轮下拉或子页面）

| 设置项 | 路由 | 说明 |
|--------|------|------|
| 公众号管理 | `/settings/accounts` | 微信公众号 CRUD |
| 写作身份 | `/settings/agents` | Agent 配置（绑定公众号 + 主题） |
| 数据源 | `/settings/sources` | RSS + 热榜平台配置 |
| 提示词库 | `/settings/prompts` | 系统模板 + 自定义模板管理 |
| LLM 配置 | `/settings/llm` | 模型选择与 API 密钥 |
| Credits | `/settings/credits` | 余额、套餐、消耗记录 |

### 2. 创作控制台 (`/create`)

**定位**：统一的创作与编辑工作台（合并了原 Create.vue 和 ArticleEditorPage.vue）

> 核心决策：不再区分"AI 创作"和"文章编辑"两个页面。
> 从文章管理点击"编辑" → 直接跳转 `/create?article_id=xxx`，在同一个三列布局中完成所有操作。

```
┌─────────────┬───────────────────────────┬──────────────┐
│ 左栏 260px   │ 中栏 自适应                │ 右栏 320px    │
│              │                           │              │
│ ▼ 写作配置   │  AI 操作栏                 │  微信预览     │
│ Agent:       │  [AI 帮我写] 基于:热点标题  │  ┌──────────┐│
│ [▼ 锐评官]   │                           │  │ 📱 手机壳 ││
│ (默认内置)    │  创作指令（用户提示词）     │  │          ││
│              │  ┌─────────────────────┐  │  │ wenyan   ││
│ 风格:        │  │ 重点分析技术突破...   │  │  │ 渲染预览  ││
│ [▼ 深度分析] │  └─────────────────────┘  │  │          ││
│              │                           │  └──────────┘│
│ 模板:        │  TipTap 编辑器             │              │
│ [▼ 爆款二创] │  ┌─────────────────────┐  │              │
│              │  │ B I U H1 H2 img ... │  │              │
│ ▼ 素材参考   │  │                     │  │              │
│ [已选素材列表]│  │ 正文编辑区域...      │  │              │
│              │  │                     │  │              │
│ ▼ 最近草稿   │  └─────────────────────┘  │              │
│ · 草稿1...   │                           │              │
│ · 草稿2...   │  [💾 保存] [📤 同步到公众号]│              │
└─────────────┴───────────────────────────┴──────────────┘
```

**默认 Agent "锐评官"**：
- 系统启动时自动创建，无需用户手动配置
- 风格定位：犀利独到的科技观察视角，善于拆解本质、观点鲜明
- 自带高质量系统提示词，确保 AI 输出有辨识度
- 用户可随时修改名称、风格、提示词，或创建自己的 Agent

**提示词融合逻辑**：
```
最终 Prompt = Agent 系统提示词（"锐评官"风格描述）
            + 风格预设（如"深度分析"的格式要求）
            + 提示词模板（如"爆款二创"的任务指令）
            + 用户创作指令（用户在输入框里写的内容）
            + 素材内容（热点标题、摘要、原文）
```

**wenyan 渲染**：
- 右栏微信预览使用 wenyan 渲染后的 HTML，确保排版与公众号一致
- 后端保存时自动用 wenyan 将 Markdown 渲染为带样式的 HTML

**关键交互**：
- 从发现页带入素材 → 左栏自动填充
- 点击"生成" → 中栏 AI 写作区流式输出
- 生成完成 → 自动切换到编辑器模式
- 右栏实时预览微信排版效果

## 一键创作弹窗 (CreateDialog)

从发现页热点卡片触发：

```
┌───────────────────────────────────────┐
│            快速创作                     │
│                                       │
│ 素材：DeepSeek 发布新一代推理模型 R2    │
│ 来源：微博热搜 #1                      │
│                                       │
│ 写作身份                               │
│ ┌─────────────────────────────────┐   │
│ │ ▼ 锐评官（内置默认）              │   │
│ └─────────────────────────────────┘   │
│                                       │
│ 创作模式                               │
│ ● 爆款二创   ○ 热点总结   ○ 观点扩写  │
│                                       │
│ 创作指令                               │
│ ┌─────────────────────────────────┐   │
│ │ 重点分析技术突破点，对比GPT-5... │   │
│ └─────────────────────────────────┘   │
│ 系统会自动融合 Agent 风格 + 模板 +     │
│ 你的指令来生成内容                      │
│                                       │
│ 预估消耗：10 Credits                   │
│                                       │
│    [ 取消 ]        [ 🚀 开始生成 ]     │
└───────────────────────────────────────┘
```

**默认行为**：
- 如果用户没有自建 Agent，自动使用内置的"锐评官"
- 如果只有一个 Agent，不显示选择器，直接使用
- "创作指令"是用户提示词，与 Agent 的系统提示词、选中的模板组合后发送给 LLM
