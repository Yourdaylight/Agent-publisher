# 交互流程：发现 → 创作

> 最后更新：2026-04-04

## 流程概述

这是 Agent Publisher 最核心的用户路径：从热点发现到完成文章创作。
系统内置默认 Agent "锐评官"，用户无需任何配置即可开始创作。

## 关键交互细节

### 1. 热点卡片交互

**悬停效果**：
- 卡片轻微上浮 + 阴影加深
- 操作按钮区域变为可见

**点击标题**：
- 展开/折叠摘要内容
- 显示原文链接（外链图标）

**跨平台标签**：
- 显示所有上榜平台 + 排名
- 例："微博#1 · 知乎#3 · 头条#5"

### 2. 快速创作弹窗

**Agent 选择器**：
- 默认选中内置 Agent "锐评官"（系统启动自动创建，零配置可用）
- 下拉列表显示：Agent 名称 + 绑定公众号（未绑定的显示"未绑定公众号"）
- 如果用户只有内置 Agent，不显示选择器，直接使用
- 用户可在设置区修改内置 Agent 的风格、提示词，或创建新的

**创作模式**：
- 爆款二创：基于原文内容二次创作（默认）
- 热点总结：从多个信源综合总结
- 观点扩写：基于热点展开个人观点

**Credits 预估**：
- 实时显示本次操作消耗
- 余额不足时按钮置灰 + 显示"余额不足"

### 3. 生成过程

**流式输出**：
- AI 生成内容逐字出现在中栏
- 顶部显示进度状态："正在生成标题..." → "正在生成摘要..." → "正在生成正文..."
- 生成期间禁止编辑，但可点击"取消生成"

**生成失败**：
- 显示错误信息 + "重试"按钮
- Credits 自动退回

### 4. 编辑模式

**切换时机**：AI 生成完成后自动切换

**AI 辅助**：
- 选中文本 → 浮动工具条 → "AI 改写"
- 改写消耗 2 Credits，改写前确认

**自动保存**：
- 每 30 秒自动保存草稿
- 顶部显示"已保存"状态

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| Credits 不足 | 弹窗提示，引导购买 |
| AI 生成失败 | 显示错误 + 重试，退回 Credits |
| 网络断开 | 本地缓存草稿，恢复后同步 |
| 重复创作同一热点 | 提示"已有相关草稿"，可选继续或查看 |

## 数据传递

```
发现页 → 弹窗
  hotspot_id, title, summary, original_url, platform

弹窗 → 创作控制台（URL参数 + localStorage）
  hotspot_id, agent_id, mode(rewrite/summary/expand), user_prompt

创作控制台 → API
  POST /api/hotspots/{hotspot_id}/create-article
  {
    agent_id: 1,          // 默认为内置 Agent "锐评官"
    mode: "rewrite",
    user_prompt: "重点分析技术突破点",  // 用户输入的创作指令
    style_preset_id: null  // 使用 Agent 默认风格
  }
```
